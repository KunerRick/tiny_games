# Tiny Vanguard — 回合制战棋肉鸽设计文档

**日期**: 2026-05-28
**版本**: v1.0
**状态**: 设计中

---

## 1. 游戏概述

**Tiny Vanguard** 是一款回合制战棋肉鸽游戏，属于 Tiny Games 合集。玩家指挥一支 3 人小队，在 6×6 网格棋盘上与敌人进行回合制战术战斗。游戏融合了 Fire Emblem 风格的回合制战斗和 Slay the Spire 风格的肉鸽路线地图。

**核心设计原则**: FE 式战术操作 + 肉鸽随机性 + 渐进式内容解锁。

### 1.1 设计目标

- **易上手难精通**: 基础操作简单（移动→攻击），战术深度来自走位和技能组合
- **高重玩性**: 不同的队伍组合 + 随机升级 + 随机路线 = 每局不同体验
- **扩展性优先**: 所有内容（职业/技能/敌人/事件）均为数据驱动，后续扩展只需加配置
- **手机友好**: 触屏点选操作，无需摇杆/虚拟按键

## 2. 核心战斗系统

### 2.1 棋盘

| 属性 | 值 |
|------|-----|
| 尺寸 | **6×6** 网格 |
| 格子类型 | 空地、障碍物（后续扩展） |
| 棋盘视角 | 全棋盘俯视，可容纳于手机屏幕 |

### 2.2 回合流程

```
战斗开始 → [布阵阶段] → 1p 行动 → 2p 行动 → 3p 行动 → 敌方行动 → 回合结束
                                                                  ↓
                                                             下一回合开始
```

- **我方阶段**: 玩家依次操作 3 个单位，顺序自由选择
- **敌方阶段**: 所有敌人按固定顺序行动（AI 控制的敌人）
- **胜利条件**: 消灭所有敌人
- **失败条件**: 我方单位全灭

### 2.3 单位每回合操作

```
1. 选中一个我方单位
2. 点击目标格子移动（格子高亮显示可移动范围）
3. 移动后，出现在攻击范围内的敌人高亮
4. 点击敌人 → 普通攻击
   或 点击技能按钮 → 选择目标 → 释放技能
   或 点击等待 → 结束该单位行动
5. 切换到下一个单位（或敌方回合）
```

### 2.4 布阵阶段

- 战斗开始前，玩家将 3 个单位自由放置在己方半场（棋盘前 2 行）
- 确认后开始战斗
- 布阵时能看到敌方阵型

### 2.5 能量系统

| 属性 | 值 |
|------|-----|
| 能量上限 | ⚡5 |
| 每回合回复 | ⚡2 |
| 普通攻击消耗 | 0 |
| 技能消耗 | ⚡1~⚡3 |

- 每个单位独立能量
- 能量在战斗中积累和使用，跨战斗重置

## 4. 肉鸽关卡系统

### 4.1 路线地图

采用经典肉鸽树形路线图：

```
                      🏆 Boss
                     /      \
              🔥精英2      🏪商店
             /      \      /
      💤休息    🔥精英1  🧪事件
        |        \       /
      ⚔️战斗4    ⚔️战斗5  ⚔️战斗6
      /    \        |       \
  🧪事件  🏪商店   ⚔️战斗3  💤休息
     \      /        \       /
     ⚔️战斗2        ⚔️战斗1
         \            /
       🏁 开局（布阵）
```

- 全图纵向滚动查看
- 当前行节点解锁后，点击下一行可达节点进入
- 完成节点后标记为已打

### 4.2 节点类型

| 节点 | 图标 | 内容 |
|------|------|------|
| ⚔️ 普通战斗 | 剑图标 | 遭遇敌人小队，胜利后获得金币 + 三选一升级 |
| 🔥 精英战斗 | 火图标 | 更难的敌人，奖励稀有技能/大量金币 |
| 🏪 商店 | 钱袋 | 花费金币购买技能/回血 |
| 💤 休息点 | 火堆 | 全员回血（可选项） |
| 🧪 事件 | 问号 | 随机事件（抉择或纯随机） |

### 4.3 升级机制

每次普通战斗胜利后，**三选一**：

- 选择一个技能分配给某个单位（从该职业的技能池中随机抽取 3 个）
- 偶尔出现：数值 Buff（+HP / +攻击 / +能量上限 / +能量回复）

**技能槽位规则**:
- 每个单位有 1 个起始技能 + 2 个升级槽位 = **最多 3 个技能**
- 如果升级槽位未满，新技能直接填入
- 如果槽位已满，需选择替换掉一个现有技能（起始技能也可以被替换）
- 替换后原技能消失

### 4.4 存档

- 每场战斗结束后自动存档
- 退出后可从最后一场战斗之前重进
- 存档只保留当前进行中的一局

## 5. 敌人设计

### 5.1 P0 四种敌人（镜像玩家职业）

| 敌人 | 对应职业 | HP | 攻击 | 防御 | 移速 | 射程 | 特殊技能 |
|------|---------|-----|------|------|------|------|---------|
| 🟢 剑兵 | 战士 | 5~8 | 3~5 | 1~2 | 3 | 近战 | 破甲斩（1.5 倍伤害） |
| 🔵 弩手 | 弓箭手 | 3~5 | 3~4 | 0~1 | 3 | 2 | 速射（连射两箭） |
| 🟤 盾兵 | 骑士 | 10~13 | 2~3 | 3~4 | 2 | 近战 | 盾墙（相邻友方受伤 -1） |
| ⚫ 影刺 | 刺客 | 4~5 | 4~5 | 0 | 4 | 近战 | 暗影步（瞬移到目标背后） |

### 5.2 精英强化版

| 精英 | 强化技能 |
|------|---------|
| 精英剑兵 | 剑气斩（直线 2 格 AOE） |
| 精英弩手 | 狙击（2 倍伤害，无视防御） |
| 精英盾兵 | 嘲讽（强制相邻玩家单位攻击他） |
| 精英影刺 | 连刺（攻击两次） |

### 5.3 Boss

- 独立模型 + 特有技能组合
- Boss 会携带 2~3 个护卫
- P0 Boss：**战争领主** — HP 20、攻击 7、防御 3、移速 3
  - 技能「横扫」: 前方 3 格直线 AOE，每 3 回合一次
  - 技能「集结」: 被动，每 2 回合召唤 1 个剑兵增援

### 5.4 AI 行为

- 优先级: 攻击范围内血量最低的玩家单位
- 各类型有差异化行为:
  - 剑兵: 直线走向最近的玩家单位
  - 弩手: 保持射程距离攻击
  - 盾兵: 优先站位保护相邻友方
  - 影刺: 绕后攻击脆皮单位

### 5.5 难度曲线

| 阶段 | 关卡 | 敌人构成 | 数量 | 敌人强度 |
|------|------|---------|------|---------|
| 🟢 前期 | 战斗 1~2 | 剑兵 + 弩手 | 2 个 | 基础属性 ×1.0 |
| 🟡 中期 | 战斗 3~5 | 加入盾兵 | 3 个 | 基础属性 ×1.2~1.4 |
| 🟠 后期 | 战斗 6~8 | 加入影刺 | 3~4 个 | 基础属性 ×1.5~1.8 |
| 🔴 精英 | 精英 ×2 | 精英替换普通兵 | 3~4 个 | 基础属性 ×2.0 |
| 🔥 Boss | 最终 | Boss + 护卫 | 3~4 个 | Boss 特有属性 |

### 5.6 敌人不使用能量

敌人不引入能量系统，每回合固定：**移动 → 攻击**。精英/Boss 的技能有固定冷却（如每 3 回合一次）。

| 精英战斗胜利 | +15~20 金币 |

| 事件奖励 | 不定 |

| 消费 | 价格 |
|------|------|
| 商店买技能 | 10~20 金币 |
| 商店回血 | 5 金币/回 2 血 |

### 6.2 局外成长（双轨制）

#### 🏅 成就系统 — 条件解锁（大目标）

| 成就 | 条件 | 奖励 |
|------|------|------|
| 🗡️ 剑术大师 | 用战士通关 1 次 | 解锁骑士职业 |
| 💀 暗影之道 | 完成 3 次不同队伍通关 | 解锁刺客职业 |
| 🔥 战争领主 | 首次击败 Boss | 解锁"困难模式" |
| 🌟 全明星 | 用 3 个不同职业通关 | 新技能加入所有职业的技能池 |

#### ⭐ 荣誉天赋树 — 累积消费（小确幸）

荣誉值获取:
| 来源 | 数量 |
|------|------|
| 通关 | +100 |
| 击杀 Boss | +30 |
| 每打过 1 关 | +5 |
| 首次用某职业通关 | +20 |

消耗 50 荣誉 = 1 天赋点，天赋树如下（每项有限购次数，防止过强）：

| 天赋 | 类型 | 效果 | 限购 |
|------|------|------|------|
| 力量强化 | 攻击 | 开局全员 +1 攻击力 | 1 次 |
| 能量充盈 | 攻击 | 开局全员 +1 能量上限 | 2 次 |
| 职业精通 | 攻击 | 某职业 +2 初始攻击 | 1 次 |
| 生命强化 | 防御 | 开局全员 +1 HP | 3 次 |
| 铁甲 | 防御 | 开局全员 +1 防御 | 1 次 |
| 能量再生 | 防御 | 每回合能量回复 +1 | 1 次 |
| 启动资金 | 经济 | 开局 +5 金币 | 3 次 |
| 议价 | 经济 | 商店折扣 -10% | 2 次 |
| 幸运儿 | 经济 | 事件好结果概率 +20% | 1 次 |

### 6.3 战后复活

- 单位在战斗中 HP 归零即阵亡，本场战斗无法再使用
- 战斗胜利后，所有阵亡单位满血复活
- 不影响下一场战斗

## 8. 数据驱动架构

所有游戏内容通过数据配置驱动，核心逻辑通用：

```typescript
// 职业配置
interface UnitClassConfig {
  id: string;
  name: string;
  baseStats: Stats;
  energyConfig: EnergyConfig;
  startingSkillId: string;
  skillPool: string[];
}

// 技能配置
interface SkillConfig {
  id: string;
  name: string;
  type: 'active' | 'passive';
  energyCost?: number;
  targetType: TargetType;
  effects: Effect[];
  triggerCondition?: TriggerCondition; // 仅被动
}

// 敌人配置
interface EnemyConfig {
  id: string;
  name: string;
  stats: Stats;
  abilityIds: string[];
  aiBehavior: AIType;
}

// 事件配置
interface EventConfig {
  id: string;
  type: 'choice' | 'random';
  choices?: ChoiceOption[];
  randomOutcomes?: RandomOutcome[];
  weight: number;
}
```

**扩展方式**:
- 新职业: 新增配置 + 素材
- 新技能: 新增配置 + 实现 Effect 逻辑（如果是新效果类型）
- 新敌人: 新增配置 + AI 行为（如果是新行为）
- 新事件: 新增配置

## 9. 技术预研重点

以下系统需要在实现前确认技术方案：

1. **网格系统**: 6×6 网格渲染、单位占格、高亮（可移动/可攻击）
2. **点击交互**: 选中单位 → 显示可选范围 → 点击目标
3. **AI 路径寻路**: 网格上的 A* 寻路
4. **战斗存档**: 序列化当前战斗状态
5. **路线图 UI**: 节点连接线、滚动、解锁逻辑
6. **技能效果系统**: Effect 的组合与执行
