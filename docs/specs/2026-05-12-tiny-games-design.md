# Tiny Games 小游戏集合 - 项目概述

**日期**: 2026-05-12  
**版本**: v1.1  
**状态**: 进行中

---

## 1. 项目概述

### 1.1 目标
开发一个休闲益智类小游戏集合，采用"容器+独立游戏"的架构模式。支持发布到微信小游戏和移动端 App。

### 1.2 核心需求
- 多游戏集合，独立场景模式
- 支持 2D 游戏，架构预留 3D 扩展能力
- 微信小游戏优先，支持 App 打包
- 资源分包加载，控制首包体积

### 1.3 目标用户
- 微信小游戏玩家（主要）
- 移动端休闲游戏用户

## 2. 技术选型

### 2.1 引擎选择: Cocos Creator 3.x

| 维度 | 评估 |
|------|------|
| 2D 支持 | ⭐⭐⭐⭐⭐ 成熟的 2D 系统 |
| 3D 支持 | ⭐⭐⭐⭐⭐ 与 2D 统一架构，可无缝扩展 |
| 微信小游戏 | ⭐⭐⭐⭐⭐ 官方支持，一键发布 |
| 跨平台 | ⭐⭐⭐⭐⭐ iOS/Android/H5/小游戏全支持 |
| 开发体验 | ⭐⭐⭐⭐⭐ TypeScript 支持，组件化开发 |
| 社区生态 | ⭐⭐⭐⭐ 中文社区活跃，文档完善 |

**选择理由**:
1. 微信小游戏原生支持，无需额外适配
2. 2D/3D 统一架构，后续添加 3D 游戏无需换引擎
3. 支持资源分包，适合游戏集合场景
4. TypeScript 开发体验好，类型安全

### 2.2 备选方案对比

| 方案 | 劣势 | 未选原因 |
|------|------|---------|
| Godot 4 | 微信小游戏支持弱 | 首要平台支持不佳 |
| Phaser | 纯 2D，无 3D 能力 | 不满足 3D 扩展需求 |
| Unity | 包体大，小游戏支持一般 | 太重，不符合轻量定位 |

## 3. 整体架构

### 3.1 架构模式

**独立模式**：每个游戏是完全独立的场景，通过主入口（游戏大厅）进入，切换时重新加载。

```
┌─────────────────────────────────────────┐
│           游戏大厅 (Lobby)               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│  │ 游戏图标 │ │ 游戏图标 │ │ 游戏图标 │   │
│  │  2048   │ │ 贪吃蛇  │ │ [更多]  │   │
│  └────┬────┘ └─────────┘ └─────────┘   │
└───────┼─────────────────────────────────┘
        │ 加载游戏场景
        ▼
┌─────────────────────────────────────────┐
│        游戏场景 (独立场景)                │
│         完全独立的场景                   │
│  ┌─────────────────────────────────┐   │
│  │         游戏逻辑层               │   │
│  │  ┌─────────┐    ┌─────────┐    │   │
│  │  │ 游戏核心 │    │ 状态管理 │    │   │
│  │  └─────────┘    └─────────┘    │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

### 3.3 扩展性设计

#### 游戏配置化

每个游戏通过配置对象注册到系统中：

```typescript
interface GameConfig {
    id: string;              // 游戏唯一标识
    name: string;            // 游戏名称
    icon: string;            // 图标资源路径
    sceneName: string;       // 场景名称
    description?: string;    // 游戏描述
}
```

#### 资源分包策略

- **主包**: 只包含大厅和公共资源，控制在 4MB 以内
- **游戏分包**: 每个游戏独立分包，按需加载（后续实现）

## 5. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 微信小游戏分包加载问题 | 高 | 提前测试分包加载，准备降级方案 |
| 触摸滑动识别不准确 | 中 | 充分测试，设置合理的滑动阈值 |
| 动画性能问题 | 中 | 使用对象池，控制同时动画数量 |
| 后续 3D 游戏复杂度 | 中 | 预留 3D 架构，但初期专注 2D |
