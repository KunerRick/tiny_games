# 2048 游戏 UI 重新设计文档

## 概述

优化 2048 游戏的设置面板和游戏结束面板，提升空间利用率和交互清晰度。

## 设计变更

### 1. 设置面板 (SettingsPanel)

#### 变更前
- 使用 5 个按钮横向排列选择网格大小 (4, 5, 6, 7, 8)
- 占用空间较大

#### 变更后
- 使用滑条 (Slider) 选择网格大小
- 范围：4-8，步长 1
- 显示当前选中的数值

### 2. 游戏结束面板 (GameOverPanel)

#### 变更前
- 信息展示单一
- 按钮层级不明确

#### 变更后
- 清晰的标题"游戏结束"
- 大号字体显示最终分数
- 主按钮"再玩一次"视觉突出
- 次按钮"返回大厅"弱化显示

## 脚本变更

### SettingsPanel.ts 变更

#### 属性变更
```typescript
// 移除
@property([Button])
gridSizeButtons: Button[] = [];

// 新增
@property(Slider)
gridSizeSlider: Slider | null = null;

@property(Label)
gridSizeValueLabel: Label | null = null;
```

#### 逻辑变更
- 监听滑条 `Slider.EventType.SLIDE` 事件
- 将滑条值 (0-4) 映射到网格大小 (4-8)
- 实时更新 `gridSizeValueLabel` 显示

### 新增 GameOverPanel.ts

```typescript
@ccclass('GameOverPanel')
export class GameOverPanel extends Component {
    @property(Label)
    scoreValueLabel: Label | null = null;
    
    @property(Button)
    restartButton: Button | null = null;
    
    @property(Button)
    backButton: Button | null = null;
    
    show(score: number, onRestart: () => void, onBack: () => void): void
    hide(): void
}
```

### Game2048.ts 变更

- 移除 `gameOverPanel` 的直接控制逻辑
- 添加 `GameOverPanel` 组件引用
- 游戏结束时调用 `gameOverPanel.show()`

## 交互规范

### 设置面板
1. 打开时滑条默认显示当前网格大小
2. 拖动滑条时数值实时更新
3. 滑条释放时自动吸附到最近刻度
4. 点击"取消"关闭面板，不应用更改
5. 点击"开始游戏"应用更改并重新开始游戏

### 游戏结束面板
1. 游戏结束时淡入显示
2. 分数数字有缩放动画效果
3. 点击"再玩一次"重新开始游戏
4. 点击"返回大厅"返回游戏大厅
